# Análisis de Sistema de Reservas – Complejo de Cabañas

Trabajo académico de análisis de sistemas de información aplicado a un caso 
de estudio real: un complejo de cabañas que gestiona reservas por múltiples 
canales (web, teléfono, presencial, Booking.com).

## Objetivo
Modelar los requerimientos y el comportamiento de un sistema de reservas, 
aplicando técnicas de análisis orientado a objetos (UML).

## Contenido
- 📋 Relevamiento de requerimientos funcionales y no funcionales
- 👥 Identificación de actores y sus roles
- 🔄 Diagrama de casos de uso
- 📝 Especificación detallada de casos de uso
- 🏗️ Diagrama de clases del dominio del problema
- 🎨 Prototipo de interfaz (mockups) de los flujos principales

## Tecnologías / técnicas aplicadas
- UML (casos de uso, clases)
- Ingeniería de requerimientos
- Prototipado de interfaces

## Contexto
Trabajo Práctico Integrador — Cátedra Análisis de Sistemas, 
Universidad Tecnológica Nacional (UTN), Facultad Regional Córdoba.
